What If?

What if I tell you any person can be identified based on the usage of a mobile phone or computer or similar kinds of devices?

Password security has been an important part of data security, and it has remained the same (only a few policy changes) despite a lot of studies, the application of new concepts, and research on futuristic concepts like password-less authentication (no one knows when it will arrive and how it will be). Complex password combinations, lengthy passwords, along with other policies have been proposed, but how an individual feels about password burden has not been explored yet.

In spite of myriad works on different aspects and modifications of password security practices, none of those previous works have talked about finding out an individual's complete password behavior. To my understanding, if the system gets to know about an individual properly, then it will be able to identify that person perfectly. That would have been possible to find out if there was an implementation that had thorough knowledge about an individual.

Although, through leaked password datasets, people tried to have a rough understanding about the masses, individual behavior could not have been studied. And this gap has been addressed by my new concept, where I proposed a complete password security mechanism that aims at building a centralized authentication mechanism that not only fulfills this gap but also many others.

Not only human behavior but also data about personal belongings, geography, and network have also been taken to assist in the identification process. This will enable the security framework to create user profiles and have a detailed understanding of their password behavior and provide security with elaborated knowledge. In that, I have been able to find 168 features so far with my analysis, trying to cover as many as possible vectors that could be useful in the classification of benign and malicious users. Among those features, there are some features that act as classes that provide important decisions about an individual.

As of now, I am talking about only the authentication mechanism, but in the future, its identification mechanism could be used in device-stealing prevention as well. If a system can bind a device to a user, then that system can know that the device is not being operated by the original owner. And the more crazy part is that the system will be able to identify who is using that device. Just imagine, by utilizing this ability, the security can go to which extent—where else can we use this mechanism!

In that framework, along with 173 features, there are the Password Dissection Mechanism and Dynamic Password Policy Mechanism that enhance the security significantly. And so far, I talked about applying machine learning models to our system, and in the future, artificial intelligence will also be implemented in that, and we believe the whole mechanism of identification will reach an unprecedented level.
